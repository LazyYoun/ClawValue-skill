## 一、项目概述

### 1.1 产品定位

ClawValue 是一个 OpenClaw 使用深度评估工具，通过采集用户的日志、技能配置、会话数据，生成趣味化的「龙虾能力估值」报告。

**核心价值**：
- 让用户直观了解自己对 OpenClaw 的使用深度
- 趣味化表达激发用户分享欲望
- 吸引更多用户使用 OpenClaw

### 1.2 目标用户

- OpenClaw 用户（主要）
- 对 AI 自动化工具感兴趣的开发者
- 喜欢分享趣味测试结果的用户

### 1.3 核心指标

| 指标 | 目标 |
|------|------|
| 页面加载时间 | < 2s |
| 分享率 | > 30% |
| 用户停留时间 | > 30s |
| 复访率 | > 20% |

## 二、核心功能需求

### 2.1 数据采集模块

#### 2.1.1 日志解析

**需求描述**：解析 OpenClaw 日志文件，提取使用统计

**数据源**：
- 路径: `/tmp/openclaw/openclaw-YYYY-MM-DD.log`
- 格式: JSONL（每行一个 JSON 对象）

**日志字段定义**：

```json
{
  "0": "[tools] 消息内容",
  "_meta": {
    "logLevelName": "INFO",
    "logLevelId": 1,
    "runtime": "node",
    "runtimeVersion": "23.5.0",
    "date": "2026-03-12T17:00:04.080Z"
  },
  "time": "2026-03-13T01:00:04.080+08:00"
}
```

**采集指标**：
- 总日志条目数
- 日志级别分布（INFO/WARN/ERROR）
- 工具调用次数
- 模型调用次数
- 错误数量

**参考文档**: https://docs.openclaw.ai/zh-CN/logging

#### 2.1.2 技能扫描

**需求描述**：扫描用户的自定义技能

**数据源**：
- 路径: `~/.openclaw/workspace/skills/`
- 文件: `SKILL.md`

**采集指标**：
- 技能总数
- 自定义技能数
- 技能分类分布
- 高风险技能识别

#### 2.1.3 配置分析

**需求描述**：解析 OpenClaw 配置文件

**数据源**：
- 路径: `~/.openclaw/openclaw.json`

**采集指标**：
- 主模型名称
- Agent 数量
- 渠道列表
- 心跳检测状态

### 2.2 评估引擎

#### 2.2.1 使用深度评估

**评分维度**：

| 维度 | 权重 | 计算规则 |
|------|------|----------|
| 技能得分 | 50% | 技能数量 × 5 + 自定义技能 × 10 |
| 自动化得分 | 30% | 心跳开启 +20，定时任务数 × 5 |
| 集成得分 | 20% | 渠道数 × 10 |

**等级划分**：

| 等级 | 名称 | 分数范围 |
|------|------|----------|
| Lv.1 | 🐣 入门小白 | 0-20 |
| Lv.2 | 🎮 初级玩家 | 21-40 |
| Lv.3 | 💻 中级开发者 | 41-60 |
| Lv.4 | 🚀 高级工程师 | 61-80 |
| Lv.5 | 🦞 龙虾大师 | 81-100 |

#### 2.2.2 价值估算

**计算公式**：
```
价值 = 基础价值 + 技能价值 + 自动化价值
- 基础价值: 100元
- 技能价值: 技能数 × 200元
- 自动化价值: 心跳开启 × 500 + 定时任务 × 100
```

#### 2.2.3 成就系统

**成就定义**：

| ID | 名称 | 条件 | 描述 |
|------|------|------|------|
| skill_master | 🏅 技能大师 | 自定义技能 ≥ 5 | 你是个真正的创造者！ |
| automation_pro | 🤖 自动化达人 | 心跳开启 | 你的AI 24小时待命！ |
| multi_channel | 🌐 多渠道运营 | 渠道 ≥ 2 | 你的AI帝国正在扩张！ |
| power_user | ⚡ 超级用户 | 日均 Token ≥ 50000 | 效率怪兽！ |

#### 2.2.4 趣味化表达

**调侃话术库**：

按等级生成不同风格的调侃文案：

- **Lv.1**: 幽默调侃，鼓励探索
- **Lv.2**: 积极引导，提示进阶

## 六、设计规范

### 6.1 常量定义原则

**单一真相源**: 所有常量集中在 `lib/constants.py`

```python
# 示例
class LogField:
    MESSAGE = '0'
    META = '_meta'
    TIMESTAMP = 'time'
```

### 6.2 数据模型原则

**类型安全**: 使用 dataclass 定义模型

```python
@dataclass
class LogEntry:
    message: str
    level: str = LogLevel.INFO
    
    @classmethod
    def from_openclaw_json(cls, data: Dict) -> 'LogEntry':
        ...
```

### 6.3 模块化原则

**职责分离**:
- `constants.py` - 常量定义
- `schemas.py` - 数据模型
- `collector.py` - 数据采集
- `evaluation.py` - 评估引擎
- `image_generator.py` - 图片生成

### 6.4 注释规范

**文档字符串**: 模块、类、方法都需要注释

```python
def calculate_rank_percentile(self, score: float) -> Dict:
    """
    计算排名百分比
    
    Args:
        score: 用户分数
        
    Returns:
        包含排名信息的字典
    """
```
